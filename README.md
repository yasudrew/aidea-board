# SpeakUp（使い捨てリアルタイム・声集めボード）

テーマ（お題でも「〇〇について考える」「〇〇部屋」でも自由）を決めて「部屋」を立ち上げると、参加者用と管理者用のリンクが発行される。
参加者は匿名で短文を投稿、管理者は付箋をドラッグして整理できる。配置はリアルタイムで全員に同期される。

- **構成**: Supabase（Postgres + Realtime + RPC）+ 静的HTML（ビルド不要）
- **アカウント不要**: 部屋コード＋管理者トークン（推測不能リンク）で保護
- **使い捨て**: 部屋は都度作成。終わったら「部屋を閉じる」

```
[index.html] 部屋を作る ─┬─ 参加リンク → [post.html]  入力 ⇄ ボード閲覧（同期）
                          └─ 管理リンク → [board.html] ドラッグ可能ボード＋操作
```

## 画面
| ファイル | 役割 |
|---|---|
| `index.html` | 部屋を作る。参加/管理リンク・QRを発行 |
| `post.html?room=コード` | 参加者。コメント入力とボード閲覧をタブ切替 |
| `board.html?room=コード&key=トークン` | 管理者。ドラッグ移動・非表示・リセット・CSV・部屋を閉じる |

## セットアップ（初回のみ）

### 1. Supabase プロジェクトを作る
1. https://supabase.com/ で無料アカウント作成 → 「New project」（リージョンは Tokyo 推奨）。
2. 左メニュー **SQL Editor** を開く。
3. [`supabase/schema.sql`](supabase/schema.sql) を全部貼り付けて **Run**。
   - `rooms` / `posts` テーブル、RLS、管理用RPC、Realtime配信が一括で設定される。

### 2. 接続情報を設定する
1. **Project Settings → API** で `Project URL` と `anon public` キーをコピー。
2. [`assets/config.js`](assets/config.js) の `SUPABASE_URL` と `SUPABASE_ANON_KEY` を置き換える。

### 3. ローカルで動作確認
```bash
cd "work/05_内製ツール_参加型体験/app/想像力ボード"
python3 -m http.server 5500
```
http://localhost:5500/ を開く → 部屋を作る → 発行された参加/管理リンクを別タブで開いて、投稿・ドラッグ・同期を確認。

### 4. Vercel に一時公開
```bash
npm i -g vercel
cd "work/05_内製ツール_参加型体験/app/想像力ボード"
vercel && vercel --prod
```
発行された `https://<app>.vercel.app/` がそのまま「部屋を作る」入口になる。

> ⚠️ 研修後は Vercel を非公開化（または削除）する。古い部屋データを消す場合は Supabase の `posts` / `rooms` を削除。

## 当日の使い方
1. 管理者が `https://<app>.vercel.app/` を開き、テーマを入れて **部屋を作る**。
2. 表示された **参加リンク（QR）** を参加者に共有。**管理者リンクは共有しない**。
3. 管理者は **管理ボードを開く** → 画面共有。投稿が届いたら付箋をドラッグして分類。
4. 必要に応じて付箋の **✕（非表示）**、**全リセット**、**CSV保存**。
5. 終了したら **部屋を閉じる**（参加者は投稿不可になる）。

## 設計メモ
- **管理操作の保護**: 非表示・移動・リセット・閉じるは、管理者トークンを検証する
  Postgres関数（RPC, SECURITY DEFINER）経由でのみ実行。参加者(anon)はRLSにより
  投稿(insert)と閲覧(select)だけ可能。直接のupdate/deleteは不可。
- **同期**: 付箋座標は正規化(0..1)でDB保存し、画面サイズが違っても相対位置を保つ。
  管理者がドラッグして指を離した時に1回だけ保存 → Realtimeで全員に反映。
- **部屋を閉じた通知**: トークン保護のため rooms はRealtime配信せず、管理者→参加者へ
  broadcast チャンネルで「閉じた」ことだけを伝える。
- **既知の制限**: anonは全部屋の `posts` を技術的には読める（部屋コードを知らなければ実害は小さい）。
  外部に長期公開せず、研修時間内の利用＋終了後の非公開化で運用カバーする。
